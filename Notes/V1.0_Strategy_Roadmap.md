# V1.0 策略開發路線圖 (V1.0 Strategy Development Roadmap)

**當前目標 (Objective)**：利用已優化的高效能 V1 引擎 (價格/成交量/時間)，立即部署可獲利的交易策略。
**限制條件 (Constraint)**：目前暫無買賣報價/掛單數據 (No Bid/Ask Order Book Data)。策略重心將放在「成交事實 (Trade Fact) 分析」。

## 1. 進階市場微結構分析 (Advanced Market Microstructure)

從每一筆 Tick 的雜訊中提取訊號。

*   **成交量分佈圖 (Volume Profile / VPVR)**：
    *   **概念 (Concept)**：即時累積每個價格價位的成交量。
    *   **目標 (Goal)**：識別出「控制點 (Point of Control, POC)」以及高/低價值區 (High/Low Value Areas)。POC 往往是強力的支撐或壓力。
    *   **實作 (Implementation)**：使用 Hash Map 或陣列 (Array)，在每筆 Tick 進來時更新該價位的累積量。
*   **交易節奏/速率 (Trade Tempo / Velocity)**：
    *   **概念 (Concept)**：測量交易的「加速度」，例如「每秒成交筆數 (Trades per second)」或「每秒成交口數 (Volume per second)」。
    *   **目標 (Goal)**：偵測「突破啟動 (Breakout Initiation)」。當價格變動伴隨著極高的交易速率，通常代表是真突破 (Genuine Breakout)。
*   **主動買賣失衡 (Tick Imbalance)**：
    *   **概念 (Concept)**：計算短時間窗口內 (如 3秒、10秒)，主動買進量 (Aggressive Buy) 與主動賣出量 (Aggressive Sell) 的比例。
    *   **目標 (Goal)**：作為短線動能策略的濾網 (Filter)。

## 2. 波動率與趨勢濾網 (Volatility & Trend Filters)

防止在盤整市場中被「雙巴 (Whipsaw)」。

*   **VWAP 通道 (VWAP Bands)**：
    *   **概念 (Concept)**：以成交量加權平均價 (VWAP) 為中心，上下加減標準差 (Standard Deviation)，例如 +/- 2.0 SD。
    *   **目標 (Goal)**：
        1.  **均值回歸 (Mean Reversion)**：在 +2SD 賣出，在 -2SD 買進。
        2.  **趨勢驗證 (Trend Validation)**：當價格持續維持在 +1SD 之上，代表強勢多頭。
*   **市場混亂指標 (Market Entropy / Choppiness Index)**：
    *   **概念 (Concept)**：用數學方法計算市場的「趨勢性 (Trendiness)」vs「混亂度 (Chaos)」。
    *   **目標 (Goal)**：如果混亂度高於閥值 (Threshold)，則暫停使用趨勢跟隨策略。

## 3. 策略與模擬系統 (Strategy & Simulation) - 快速獲利路徑

建立能夠驗證獲利的基礎設施。

*   **模擬交易模組 (Paper Trading Module)**：
    *   **優先級 (Priority)：高 (HIGH)**
    *   **描述 (Description)**：一個能夠接收 `開倉 (Open)` / `平倉 (Close)` 訊號的類別 (Class)，並根據即時價格模擬撮合 (Fill)。
    *   **產出 (Output)**：即時追蹤損益 (Real-time P&L Tracking)，在不冒真金白銀風險的情況下驗證策略。
*   **市場重播器 (Market Replay / Time Machine)**：
    *   **描述 (Description)**：以 10 倍速將錄製下來的歷史 Tick 數據「倒」進共享記憶體 (Shared Memory)。
    *   **目標 (Goal)**：週末時能像看電影一樣重播行情，進行快速策略迭代 (Strategy Iteration) 與盤感訓練 (Eye Training)。

## 4. 資料工程 (Data Engineering)

*   **Tick 錄製器 (Tick Recorder)**：
    *   確保每一筆 Tick 都被完整儲存為 Parquet 或 HDF5 格式，作為未來 AI 模型訓練 (V2 Preparation) 的資產。
