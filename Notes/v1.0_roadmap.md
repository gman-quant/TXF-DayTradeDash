# V1.0 完工路線圖 (Pre-BidAsk) - 邁向 V2.0 之前

在整合 Bid/Ask 委託簿數據 (V2.0) 之前，我們應先最大化利用現有的 `Tick` 數據潛力，並強化基礎設施的強韌度。

## 1. 進階 Tick 基礎指標 (Advanced Tick-Based Indicators) - 遺失的 Alpha

目前我們只有 VWAP (成交量加權平均價) 和基礎的動能指標。我們缺少了對於「均值回歸 (Mean Reversion)」策略至關重要的 **波動率 (Volatility)** 與 **統計 (Statistical)** 指標。

### A. 波動率與統計指標 (Volatility & Stat Indicators)
*   **布林通道 / 凱特納通道 (Bollinger Bands / Keltner Channels)**:
    *   需要實作 `標準差 (Standard Deviation)` 計算。
    *   **用途 (Use)**: 識別「擠壓 (Squeeze, 低波動)」後的「突破 (Breakout, 高波動)」。
*   **Z-Score (標準分數)**:
    *   量化當前價格變動的「稀有度 (Rarity)」。
    *   **用途 (Use)**: 當 Z-Score > 2 或 < -2 時，產生均值回歸訊號。

### B. 動能震盪指標 (Momentum Oscillators) - 快速版
*   **RSI (相對強弱指標)**:
    *   標準但有效的指標。可以使用 Numba 優化至 O(1) 或 O(N) 複雜度。
*   **ROC (變動率, Rate of Change)**:
    *   純粹的價格速度 (Velocity)。

### C. 成交量分佈 (Volume Profile) - Session Logic
*   **POC (控制點, Point of Control)**: 當日 Session 中成交量最大的價格價位。
*   **VAH/VAL (價值區高點/低點, Value Area High/Low)**: 包含 70% 成交量的價格區間。
*   **為什麼 V1.0 要做? (Why V1.0?)**: 它完全可以只用 Tick (價格/量) 構建出來，不需要 Order Book。它是最強力的支撐/壓力參考。

## 2. 基礎設施升級：共享記憶體 (Infrastructure Upgrade: Shared Memory) - **關鍵任務**

目前的瓶頸：Python GIL (全域直譯器鎖)。即使 Numba 開了 `nogil=True`，核心處理器 (`CoreProcessor`) 的 Python 迴圈仍然佔用著鎖，與 Dashboard 伺服器搶資源。

### 解決方案：`SharedMemoryRingBuffer`
將目前的 `np.zeros` 陣列替換為 `multiprocessing.shared_memory.SharedMemory`。

#### 架構變更 (Architecture Change)
*   **Process A (資料接收層, Ingestion)**: 專注寫入 Shared Memory。零計算量。無限吞吐量。
*   **Process B (策略/儀表板, Strategy/Dash)**: 唯讀 (Read-Only) 存取 Shared Memory。計算指標、運行 Dash。
*   **效益 (Benefit)**: 就算 Dash 當機或計算卡住，資料錄製 **永遠不會停止**。

## 3. 「時光機」重播引擎 (The "Time Machine" Replay Engine)

我們有 `consume_history`，但我們需要一個強健的 **事件驅動回測器 (Event-Driven Backtester)**。

*   **目標 (Goal)**: 讓同一份策略代碼能跑在歷史 Parquet 檔案上。
*   **原因 (Why)**: 沒有這個，你無法優化參數 (例如 SMA 週期、進場閥值)。
